代码版本管理工具Git基本命令(基本使用)

git命令：注：<>表示占位实际命令不需要

1、git – version 查看版本信息

2、pwd 查看当前路径

3、ls 查看当前目录所有文件夹

4、mkdir 创建文件夹

5、git init 初始化本地仓库(会创建一个.git的隐藏文件夹，不可删除)

6、ls -a 查看隐藏文件夹

7、git add 添加文件至缓存区(.表示添加所有未加入缓存区的文件)

8、git status 查看状态

9、git commit -m ‘提交名称(可中文)’ 将文件提交至仓库
第一次提交时设置
(
10、 git config –global user.email “your@example.com“ 设置提交者邮箱

11、 git config –global user.name “Your Name” 设置提交者名称
)
12、git log 查看提交日志

13、git restore 用缓存区恢复工作区

14、git restore –staged 用仓库恢复暂存区

15、git log –abbrev-commit –pretty=oneline 日志同行显示

16、git reflog 查看过去和未来的日志

17、git reset –hard：让仓库、缓存区、工作区都回退到指定版本(三者一致)
–mixed：默认选项，让仓库、缓存区回退到指定版本，工作区保持现状
–soft：让仓库回退到指定版本，缓存区和工作区保持现状

18、git remote add origin 添加远端仓库

19、git remote remove origin 移除远端仓库

20、git remote -v 查看远端仓库

21、git push -u origin master 上推代码(向服务器推代码)

22、git pull 下拉(从服务器获取代码) git fetch + git merge

23、git clone –depth 1 从服务器克隆代码

24、 ssh-keygen -t rsa -b 2048 -C ‘123456@qq.com‘ 生成密钥对 ssh开头邮箱(终止符)结尾

25、notepad ~/.ssh/id_rsa.pub 以记事本打开查看密钥(cat:在命令行查看)

26、git config user.name 查看用户名

​ —– 分支操作 ——
27、git branch 新建名为name分支
28、git checkout/switch 切换到名为name分支
29、git merge –no–ff 合并名为name与当前分支，(–no-ff：禁用快速合并)
30、git branch 查看分支
31、git branch -a 查看所有分支(包括远端分支)
32、git branch -d 删除名为name的分支
33、git switch -c / git checkout -b 创建切换分支
34、变基：
​ git rebase
​ git switch
​ git merge

35、git merge –abort 取消合并

push时产生冲突解决办法：
遇到冲突如何解决
1．从服务器fetch最新的代码，放到一个临时分支

  • git fetch origin master:temp
    2．查看哪里有冲突
  • git diff temp
    3．尝试将临时分支合并到当前分支—>CONFLICT
  • git merge temp
  • 合并虽然是失败的，但是有冲突的代码已经被标记了
    ~取消合并:git merge –abort
    4．解决代码冲突(找到当事人进行沟通，确定保留谁的代码)
    5．重新提交解决冲突后的代码并上推到服务器
  • git add
  • git commit -m ‘…’
  • git push -u origin
    6．删除临时分支
  • git branch -d temp
    7．重新发起Pull Request
